# Git 運用




## 🌿 ブランチ方針

| 項目 | 決め方 |
|------|------|
| PR先 | ブランチ分岐元へ向けて PR を作成する |
| Reviewer | **@codex review** |

- PR では Reviewer に **@codex review** を指定する
- 後から `git log` を見たとき、**何の変更か一目で分かる**（下記**コミットメッセージのルール**）



### プレフィックスの使い分け

| プレフィックス | 用途 |
|----------------|------|
| `docs/` | 仕様・手順・README など |
| `feature/` | バッチ実装・機能追加・PNG・テストなどコード変更全般 |
| `fix/` | バグ修正 |
| `refactor/` | 依存・設定・構成整理（バッチの成果物・振る舞いは変えない） |

**ブランチ名のプレフィックス**（`docs/` 等）と**コミットの Type**（`Docs` / `Chore` 等）は別。


## 😀 コミットメッセージのルール

形式（すべてこの形）:

```text
{emoji} {Type}: {日本語の要約}
```

- 要約内の**英字・数字と日本語の間に半角スペースを入れない**（例: `Python実行の土台`。`Python 実行` のように挟まない）

| Type | emoji | 用途（例） |
|------|-------|------------|
| Docs | 📚 | `📚 Docs: デプロイ手順を追加` |
| Chore | 🔧 | `🔧 Chore: Python実行の土台を追加` |
| Feat | ✨ | `✨ Feat: iCal取得・解析を追加` |
| Fix | 🐛 | `🐛 Fix: 失敗時もDotを更新する不具合` |
| Update | 🎨 | `🎨 Update: PNGの今日・明日レイアウトを調整` |
| Refactor | ♻️ | `♻️ Refactor: PNG生成をモジュール分割` |
| Perf | 🐎 | `🐎 Perf: iCal取得を並列化` |
| Test | 🚨 | `🚨 Test: 予定の半開区間判定を追加` |
| Chore | 🗑️ | `🗑️ Chore: 未使用の設定キーを削除` |
| Chore | 💩 | `💩 Chore: 旧環境変数を非推奨化` |
| Chore | 🔖 | `🔖 Chore: v0.1.0 をタグ付け` |
| Wip | 🚧 | `🚧 Wip: Dot送信は未接続` |
| — | 🎉 | `🎉 Initial commit`（`main` の初回のみ。Type 省略） |

**この表以外の絵文字は使わない。**
