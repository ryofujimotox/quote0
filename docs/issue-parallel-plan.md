# Issue 並行解決プラン




## 使い方

- オープン Issue を洗い出し、下記 **テンプレート** をリポジトリ直下の `plan.md` にコピーして埋める
- Issue 表でファイル競合を整理し、PR 表でブランチ統合・マージ順・**待つ PR** を決める
- エージェントへは `plan.md` を渡して実装・PR 作成を依頼する
- Git 運用（base・`Fixes #N`・レビュー）は [git.md](git.md) に従う




## テンプレート

- PR 表の **No** は連番とし、`dev` へのマージ順とする
- **待つ PR** は、ファイル競合のため先に `dev` へ入れるべき PR の No（PR 表の列）
- 同テーマの Issue は 1 ブランチ 1 PR にまとめてよい（**Issue ごとにコミットを分ける**）



### 進め方

1. **待つ PR** があっても、PR ごとに並行で実装し、それぞれ初回レビューを依頼する
2. **待つ PR** が `dev` に入ったら、それを待っていた各 PR のブランチへ `dev` の最新を取り込み、再レビューを依頼する
3. 同じ **待つ PR** を持ち、互いにファイル競合しない PR は、再レビュー後に同時期に `dev` へマージしてよい
